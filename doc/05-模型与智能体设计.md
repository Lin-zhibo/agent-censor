# 05-模型与智能体设计

## 1. 文档口径

本文档描述多模态模型和智能体的目标设计。当前内容为建议实现，不表示模型训练、智能体编排或模型部署已经完成。

## 2. 设计目标

模型与智能体部分需要覆盖：

- 更细标签识别。
- 连续对话能力。
- 多模态审核能力。
- 工具/skill 调用。
- 模型路由和任务划分。
- Graph RAG 证据增强。
- 模型训练、部署和准确率评估。

## 3. 多模态模型能力

后续实现建议按模态拆分模型能力：

标签字段值统一使用大写英文，限制为 1-2 个 word；多个 word 使用下划线连接，例如 `SECURITY`、`PORN`、`SUPPORT_SUICIDE`。不使用连字符或中文。常见前缀如 `ECO`、`AD`、`TERROR` 按 1 个 word 计算；中文名称和说明放在展示名或描述字段中。

| 模态 | 目标能力 | 输出 |
| --- | --- | --- |
| text | 敏感文本、导流、辱骂、不良价值观检测 | 标签、分数、文本证据 |
| image | 领导人画像、旗帜图案、色情低俗、未成年风险检测 | 标签、分数、检测框 |
| video | 抽帧检测、时间段聚合、封面风险检测 | 标签、分数、命中时间段 |
| audio | ASR 转写、敏感语音、低俗内容检测 | 标签、分数、转写证据 |
| multimodal | 图文一致性、图文组合风险检测 | 标签、分数、跨模态解释 |

## 4. 模型路由

模型路由使用 `ModelRouteDecision` 记录选择结果。路由依据包括：

- `modality`：根据模态选择文本、图片、视频、音频或多模态模型。
- `business_id`：根据业务风险选择严格或通用模型。
- `policy_id`：根据策略选择模型阈值。
- 内容特征：例如文本长度、图片数量、视频时长。
- 模型可用性：主模型不可用时使用 `fallback_model`。

模型路由示例：

```json
{
  "modality": "image",
  "selected_model": "image_safety_v1",
  "reason": "图片模态审核，优先选择图片安全模型",
  "fallback_model": "vision_general_baseline"
}
```

## 5. 智能体编排

智能体不直接替代审核策略，而是负责任务编排和解释生成。后续实现应采用树型 sub-agent 体系，并以 `config/settings.json` 的 `rules` 字段作为标签树来源。

- 接收审核任务上下文。
- 从 `rules.security` 构建 `SECURITY` 子树，从 `rules.ecosystem` 构建 `ECOSYSTEM` 子树。
- RootAgent 调用 `SecurityAgent` 和 `EcosystemAgent`，中间智能体继续调用自己的子智能体，叶子智能体负责最小标签判断。
- 叶子智能体返回 `{ "label": "...", "reason": "..." }`；未命中时返回 `{ "label": "", "reason": "未检出问题" }`。
- 中间智能体返回 `{ "label": ["..."], "reason": "..." }`，其中 `reason` 是对子智能体返回原因的综合洞察。
- RootAgent 返回 `{ "security_labels": ["..."], "ecosystem_labels": ["..."], "reason": "..." }`。
- 父智能体不得修改、添加、删除收集到的子智能体标签，只能综合子智能体的 `reason`。
- 调用模型路由工具。
- 调用规则查询工具。
- 调用 Graph RAG 检索工具。
- 生成可解释审核摘要。
- 支持运营人员追问，例如“为什么判为复核”“哪个规则命中”“阈值调高会怎样”。

## 6. 连续对话与记忆

连续对话目标用于运营解释和策略调试：

- 保留当前任务的 `AuditTrace`、树型智能体各级返回、命中标签、规则和 RAG 证据。
- 对长上下文进行压缩，只保留策略、标签、证据和用户操作。
- 支持按 `trace_id` 查询历史审核上下文。
- 支持回答策略相关追问，但不得绕过规则引擎直接改变审核结论。

## 7. Graph RAG 参与审核解释

Graph RAG 应服务于“解释”和“证据增强”，不是单纯生成文本。建议流程：

1. 使用当前叶子标签、待审核文本、证据片段和业务上下文构造检索查询。
2. 在当前叶子标签邻域内召回规则说明、样本和关键词知识节点。
3. 在知识图谱中扩展相邻节点，例如叶子标签、规则、样本、关键词。
4. 返回命中节点、关系路径、相似度或置信度、证据摘要。
5. 叶子智能体将这些证据作为裁决参考，并写入 `ModerationResult.explanation` 和前端证据面板。

示例关系路径：

```text
Label:SEX_SERVICE -> Sample:SEX_SERVICE:003
```

## 8. 模型训练方案

模型训练目标是解决课程描述的多模态内容安全问题。建议训练方向：

- 色情裸露内容检测。
- 性暗示和低俗内容检测。
- 导流和辱骂内容检测。
- 未成年人安全风险检测。

训练方案建议：

- 使用 baseline 模型建立初始准确率。
- 在清洗和标注数据上微调或训练专项模型。
- 记录准确率、召回率、误杀率、漏放率和相对 baseline 改进。
- 对高风险标签优先关注召回率和可解释证据。
- 即使某些训练策略降低 AUC，也应记录其业务收益和创新点。

## 9. 模型部署方案

后续实现建议：

- 将模型封装为独立推理服务。
- API 服务通过模型路由调用模型服务。
- 支持模型版本管理和灰度切换。
- 支持主模型失败后的 fallback。
- 支持批量推理、缓存和异步任务队列。

## 10. 工具和 skill

建议智能体可调用的工具/skill：

- `model_route`：选择模型。
- `run_model_inference`：执行模型推理。
- `query_policy_rule`：查询规则和策略。
- `graph_rag_search`：仅由叶子智能体调用，检索当前叶子标签邻域内的规则、样本和关键词知识证据。
- `audit_trace_lookup`：查询历史审核轨迹。
- `threshold_preview`：预览阈值变化影响。

所有工具调用均为目标设计，后续实现建议通过 `/internal/v1` HTTP/JSON 内部 API 完成。工具响应统一包装为：

```json
{
  "status": "success",
  "data": {},
  "errors": [],
  "latency_ms": 120,
  "trace_id": "trace_20260604_0001"
}
```

错误对象统一为：

```json
{
  "code": "TOOL_TIMEOUT",
  "message": "工具调用超时",
  "retryable": true
}
```

## 11. model_route 工具

用途：根据租户、业务、策略、模态和内容特征选择模型。

内部端点：`POST /internal/v1/models/route`。

输入：

```json
{
  "trace_id": "trace_20260604_0001",
  "tenant_id": "tenant_demo",
  "business_id": "community_post",
  "policy_id": "default_policy",
  "modality": "image",
  "content_features": {
    "text_length": 0,
    "image_count": 1,
    "video_duration_sec": null,
    "language": null
  },
  "candidate_models": ["image_safety_v1", "vision_general_baseline"]
}
```

输出 `data`：

```json
{
  "modality": "image",
  "selected_model": "image_safety_v1",
  "reason": "图片模态审核，优先选择图片安全模型",
  "fallback_model": "vision_general_baseline"
}
```

错误处理：

- 默认超时 500 ms。
- 路由失败时可选择 `fallback_model`，并在 `errors` 中记录 `MODEL_ROUTE_FALLBACK`。
- 找不到可用模型时返回 `MODEL_ROUTE_NOT_FOUND`，不可重试。

## 12. run_model_inference 工具

用途：调用模型服务执行推理并返回标准化 `ModelResult`。

内部端点：`POST /internal/v1/models/infer`。

输入：

```json
{
  "trace_id": "trace_20260604_0001",
  "task_id": "task_0001",
  "route_decision": {
    "modality": "text",
    "selected_model": "text_safety_v1",
    "reason": "文本模态审核",
    "fallback_model": "text_baseline"
  },
  "content": {
    "text": "待审核文本",
    "metadata": {}
  },
  "labels_requested": ["PORN", "ECO_DIVERSION"],
  "detail_level": "detailed",
  "timeout_ms": 3000
}
```

输出 `data`：

```json
{
  "model_name": "text_safety_v1",
  "model_version": "2026-06",
  "modality": "text",
  "labels": [
    {
      "label": "PORN",
      "sub_label": "SEX_BEHAVIOR",
      "score": 0.82,
      "normalized_score": 0.82
    }
  ],
  "evidence": [
    {
      "evidence_id": "ev_text_001",
      "type": "text_span",
      "content": "命中证据片段",
      "start": 0,
      "end": 6
    }
  ],
  "latency_ms": 420,
  "status": "success",
  "error": null
}
```

错误处理：

- 单条审核默认超时 3000 ms。
- 超时返回 `MODEL_TIMEOUT`，可重试；若存在 fallback 模型，智能体应调用 fallback。
- 模型不可用返回 `MODEL_UNAVAILABLE`，可重试。
- 内容格式错误返回 `MODEL_INPUT_INVALID`，不可重试。

## 13. query_policy_rule 工具

用途：查询当前租户、业务、模态、策略版本下生效的规则。

内部端点：`POST /internal/v1/rules/query`。

输入：

```json
{
  "trace_id": "trace_20260604_0001",
  "tenant_id": "tenant_demo",
  "business_id": "community_post",
  "policy_id": "default_policy",
  "policy_version": "v1",
  "modality": "text",
  "labels": ["PORN"]
}
```

输出 `data`：

```json
{
  "rules": [
    {
      "rule_id": "rule_model_score_001",
      "label": "PORN",
      "condition": {
        "type": "model_score",
        "label": "PORN"
      },
      "threshold": 0.8,
      "action": "review",
      "enabled": true,
      "version": "v1"
    }
  ]
}
```

错误处理：

- 默认超时 500 ms。
- 策略不存在返回 `RULE_POLICY_NOT_FOUND`，不可重试。
- 规则缓存未命中可访问数据库补查，失败时返回 `RULE_QUERY_TIMEOUT`，可重试。

## 14. graph_rag_search 工具

用途：检索政策、案例、规则和样本证据，返回 Graph RAG 命中节点和关系路径。

内部端点：`POST /internal/v1/rag/search`。

输入：

```json
{
  "trace_id": "trace_20260604_0001",
  "query": "PORN SEX_BEHAVIOR",
  "labels": ["PORN"],
  "evidence": [
    {
      "evidence_id": "ev_text_001",
      "type": "text_span",
      "content": "命中证据片段"
    }
  ],
  "policy_id": "default_policy",
  "business_id": "community_post",
  "top_k": 5,
  "max_depth": 2,
  "node_types": ["Policy", "Case", "Rule", "Label"]
}
```

输出 `data`：

```json
{
  "hits": [
    {
      "node_id": "policy_P001",
      "node_type": "Policy",
      "title": "色情内容审核政策",
      "similarity": 0.87,
      "confidence": 0.81,
      "summary": "该政策说明色情裸露内容需要进入复核。"
    }
  ],
  "paths": [
    {
      "path": ["PORN", "Policy:P001", "Rule:rule_model_score_001"],
      "score": 0.84
    }
  ],
  "evidence_summary": "命中色情内容标签相关政策和规则，建议复核。"
}
```

错误处理：

- 默认超时 1000 ms。
- 向量库不可用返回 `RAG_VECTOR_UNAVAILABLE`，可重试或降级为图谱查询。
- 图谱不可用返回 `RAG_GRAPH_UNAVAILABLE`，可重试或降级为向量检索。
- 无召回结果返回 `status = success` 且 `hits = []`，不作为错误。

## 15. audit_trace_lookup 工具

用途：根据 `trace_id` 查询历史审核轨迹，用于连续对话和问题复盘。

内部端点：`GET /internal/v1/audit/traces/{trace_id}`。

输入：

```json
{
  "trace_id": "trace_20260604_0001"
}
```

输出 `data`：`AuditTrace`。

错误处理：

- 默认超时 500 ms。
- 找不到轨迹返回 `AUDIT_TRACE_NOT_FOUND`，不可重试。
- 审计库超时返回 `AUDIT_TRACE_TIMEOUT`，可重试。

## 16. threshold_preview 工具

用途：预览阈值调整对审核结论的影响，不写入正式审核结果。

内部端点：`POST /internal/v1/policies/{policy_id}/preview`。

输入：

```json
{
  "trace_id": "trace_20260604_0001",
  "task_id": "task_0001",
  "policy_version": "v1",
  "threshold_overrides": {
    "PORN": 0.9
  },
  "model_results": [],
  "rag_evidence": []
}
```

输出 `data`：

```json
{
  "decision": "pass",
  "risk_score": 0.82,
  "labels": [
    {
      "label": "PORN",
      "score": 0.82
    }
  ],
  "rule_results": [],
  "decision": "pass",
  "preview_only": true
}
```

错误处理：

- 默认超时 1000 ms。
- 阈值配置非法返回 `POLICY_PREVIEW_INVALID_THRESHOLD`，不可重试。
- 缺少模型结果返回 `POLICY_PREVIEW_MISSING_MODEL_RESULT`，可重试或等待异步补算。

## 17. 工具调用顺序

智能体处理单条审核任务时建议按以下顺序调用：

1. `model_route` 选择模型。
2. `run_model_inference` 获取标准化 `ModelResult`。
3. `query_policy_rule` 查询生效规则。
4. 调用规则执行内部接口 `/internal/v1/rules/evaluate` 获取 `RuleResult`。
5. `graph_rag_search` 获取政策、案例、规则证据。
6. 汇总模型、规则和 Graph RAG 结果，生成审核解释。
7. 需要连续对话或复盘时调用 `audit_trace_lookup`。
8. 前端调整策略阈值时调用 `threshold_preview`。
