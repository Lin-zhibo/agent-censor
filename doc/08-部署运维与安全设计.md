# 08-部署运维与安全设计

## 1. 文档口径

本文档描述 Agent Censor 的目标部署、运维和安全设计。当前内容为建议实现，不表示服务已经部署完成。

## 2. 部署目标

目标系统建议拆分部署：

- 前端服务：审核工作台和策略配置界面。
- Go API 服务：统一接口、鉴权、任务管理和结果查询。
- 规则引擎服务：规则执行、阈值判断和策略版本管理。
- 智能体服务：任务编排、工具/skill 调用、解释生成。
- 模型服务：文本、图片、视频、音频和多模态推理。
- RAG 服务：向量检索、Graph RAG 检索和证据摘要。
- 数据库服务：业务数据库、embedding 数据库、图谱数据库或关系表。
- 对象存储：图片、视频、音频和原始样本。

## 3. 配置管理

配置应按环境和业务隔离：

- 环境：开发、测试、演示、生产。
- 租户：`tenant_id`。
- 业务：`business_id`。
- 策略：`policy_id` 和 `version`。
- 模型：模型名称、版本、阈值和 fallback。
- RAG：向量库索引、图谱版本、召回参数。

敏感配置应使用安全存储，不应硬编码在代码或文档样例中。

## 4. 高并发设计

高并发目标覆盖 API、模型和数据检索：

- API 层：限流、请求队列、批量接口、超时控制。
- 模型层：批处理、异步推理、模型缓存、fallback 模型。
- 规则层：策略缓存、规则版本缓存。
- RAG 层：embedding 缓存、热门政策和案例缓存。
- 数据层：读写分离、索引优化、任务状态异步更新。

前端应对排队、处理中、超时、降级结果进行明确展示。

## 5. 监控告警

建议监控指标：

- API 请求量、错误率、P95/P99 时延。
- 审核任务成功率、失败率、排队长度。
- 模型推理耗时、超时率、fallback 次数。
- 规则命中分布、策略版本变更次数。
- Graph RAG 检索耗时、召回数量、空召回比例。
- 数据采集失败率、清洗失败率、标注进度。
- 人工复核积压量和处理时长。

告警场景：

- API 错误率异常升高。
- 模型服务不可用或超时。
- 规则配置异常。
- Graph RAG 检索不可用。
- 批量任务积压。
- 爬虫或数据清洗失败。

## 6. 权限与数据安全

权限建议按角色控制：

- 管理员：管理租户、业务、策略、规则和用户。
- 策略运营：编辑规则、阈值和处置策略。
- 审核员：查看任务、执行人工复核。
- 模型工程师：查看数据集、模型版本和评估报告。
- 只读访客：查看报告和演示结果。

数据安全要求：

- 租户和业务数据隔离。
- 审核原始内容和证据按权限访问。
- 人工复核操作写入审计。
- 敏感数据脱敏后用于展示或报告。
- 数据集按训练、验证、测试、RAG、演示用途隔离。

## 7. 爬虫与数据采集合规

爬虫和采集服务应遵守：

- 仅采集授权或公开数据。
- 遵守访问频率限制和 robots 规则。
- 记录采集来源、时间、脚本版本和用途。
- 支持限速、重试、断点续爬和失败告警。
- 不设计验证码破解、登录绕过、规避访问控制等能力。

## 8. 部署验收

部署验收建议检查：

- 前端可访问并展示审核工作台。
- Go API 服务可提交和查询审核任务。
- 规则引擎可读取策略版本并返回规则命中结果。
- 模型服务可按模态返回模型结果。
- Graph RAG 可返回命中节点、关系路径和证据摘要。
- 审计日志可追踪请求、模型、规则、RAG 和人工处置。
- 高并发场景下系统具备限流、排队、降级和告警能力。
