# 01-需求规格说明书

## 1. 文档口径

本文档描述 Agent Censor 的目标需求和建议实现范围。当前仓库为早期目录骨架，本文不声明已有代码实现已完成，后续实现应以本文作为需求验收依据。

## 2. 项目目标

Agent Censor 目标是建设一个多业务、多模态、可解释、可追踪的内容安全审核系统。系统应支持文本、图片、视频、音频等内容输入，通过规则引擎、多模态模型、智能体工具调用和 Graph RAG 知识证据，输出是否违规、风险分数、详细标签、证据片段和建议处置动作。

## 3. 审核范围

安全类风险：

- 色情低俗：裸露、性暗示、低俗擦边、成人内容导流。
- 不良价值观：暴力、仇恨、歧视、违法违规诱导、极端言论。
- 未成年安全：未成年色情、诱导未成年消费、未成年隐私泄露、校园欺凌。

生态类风险：

- 导流：私聊导流、联系方式导流、外部链接导流、非官方支付方式导流。
- 辱骂：个人歧视、贬低人格、性方面侮辱、诅咒威胁、家庭组织威胁。

## 4. 标签体系

标签体系建议采用分层结构，满足“是否违规”和“详细标签”两种输出粒度。

标签编码统一使用大写英文，限制为 1-2 个 word；多个 word 使用下划线连接，例如 `TERROR`、`SUPPORT_SUICIDE`。不使用连字符或中文。常见前缀如 `ECO`、`AD`、`TERROR` 按 1 个 word 计算；中文名称和说明应放在展示名或描述字段中。

| 0 级处置域 | 一级标签编码 | 叶子标签编码示例 | 中文说明 | 建议处置 |
| --- | --- | --- | --- | --- |
| SECURITY | PORN | FETISH, SEX_SERVICE, SEX_BEHAVIOR | 色情低俗 | reject, review |
| SECURITY | BAD_VALUES | SUICIDE_GAME, SUPPORT_SUICIDE, GENDER_OPPOSITION | 不良价值观 | reject, review |
| SECURITY | ILLEGAL_BEHAVIOR | PRIVACY_LAW, FINANCE_CRIME, CONTROLLED_TOOL | 违法行为 | reject, review |
| ECOSYSTEM | ECO_DIVERSION | PRIVATE_CHAT, EXTERNAL_LINK, PAYMENT_TRAFFIC | 导流 | review, pass_with_limit |
| ECOSYSTEM | ECO_INSULT | PERSONAL_BIAS, SEXUAL_INSULT, FAMILY_INSULT | 辱骂 | review |

后续实现应允许标签继续细分，并在 `GET /api/v1/labels` 中返回标签编码、名称、父级标签、说明、默认阈值和建议处置动作。

## 5. 功能需求

### 5.1 审核任务

- 支持单条审核和批量审核。
- 支持通过 `tenant_id` 区分公司，通过 `business_id` 区分业务线，通过 `modality` 区分内容模态。
- 支持指定 `policy_id` 和策略版本，保证不同业务可以使用不同审核尺度。
- 支持 `detail_level` 控制输出粒度，仅输出违规结论或输出详细标签、证据和解释。

### 5.2 前端审核工作台

前端目标不是简单展示结果，而是作为可交付演示和运营审核入口。后续实现应至少支持：

- 审核结果在 3 秒内呈现，超时任务显示排队、处理中或降级结果。
- 命中规则高亮显示证据片段，例如文本中的敏感词、图片中的检测框、视频中的命中时间段。
- 展示标签、风险等级、模型来源、规则来源、Graph RAG 证据摘要和最终解释。
- 支持人工复核一键通过、打回、改标、补充备注。
- 支持策略阈值调整后的审核结果预览，方便展示“处理尺度怎么调整”。

### 5.3 规则与策略

- 支持规则启用、停用、版本管理和阈值调整。
- 支持不同标签、业务、模态使用不同阈值。
- 支持处置策略配置，例如 `pass`、`review`、`reject`、`pass_with_limit`。
- 支持规则命中结果和模型结果合并，形成最终审核结论。

### 5.4 智能体能力

- 支持连续对话，用于策略解释、历史审核追问和运营辅助。
- 支持上下文压缩和记忆模块，保留关键策略、标签和审核轨迹。
- 支持工具/skill 调用，例如规则查询、Graph RAG 检索、模型推理、数据样本检索。
- 支持模型路由，根据模态、任务类型、风险等级选择合适模型。
- 支持从 `config/settings.json` 的 `rules` 字段构建树型 sub-agent，`rules.security` 映射为 `SECURITY`，`rules.ecosystem` 映射为 `ECOSYSTEM`。
- 支持叶子、中间、顶级智能体按已定义契约返回 `label`、`security_labels`、`ecosystem_labels` 和 `reason`。
- 父智能体不得修改、添加、删除收集到的子智能体标签，只能综合子智能体的 `reason`。

### 5.5 数据与爬虫

- 支持数据集、爬虫数据、人工标注数据和 RAG 知识数据管理。
- 爬虫仅采集授权或公开数据，记录来源、时间、模态、原始引用、清洗状态、标签状态和用途划分。
- 反爬设计仅包含合规限速、重试、断点续爬、失败告警和访问礼貌策略，不包含验证码破解、登录绕过或规避访问控制。

### 5.6 Graph RAG 可解释性

- 建立内容安全标签、政策条款、典型案例、规则、样本之间的知识图谱。
- 审核时结合向量检索和图谱邻域扩展，找到相关政策、案例和规则。
- 输出命中节点、关系路径、相似度或置信度、证据摘要，用于解释审核结论。

## 6. 非功能需求

- 准确性：模型效果以准确率为核心指标，并说明相对 baseline 的改进。
- 时延：常规单条审核目标 3 秒内返回可展示结果。
- 并发：后端和模型服务应支持队列、限流、批处理和降级策略。
- 可追踪：每次审核必须形成 `AuditTrace`，记录请求、树型智能体返回、模型、规则、RAG 证据和处置结果。
- 可配置：规则、阈值、策略和模型路由应支持配置化。
- 安全：对租户、业务、策略、数据和审计日志进行权限隔离。

## 7. 验收指标

- 能说明并演示标签体系覆盖 `NOTE.md` 中的安全类和生态类风险。
- 能说明并演示树型智能体从 `config/settings.json.rules` 构建标签树，并保持父智能体不修改、添加、删除子智能体标签。
- 能通过 API 提交文本或图片审核任务，并返回 `ModerationResult` 结构。
- 能展示前端审核结果、证据高亮和人工复核动作。
- 能展示规则阈值调整对审核结论的影响。
- 能说明模型路由、Graph RAG 证据链和审计追踪如何工作。
- 能提供准确率、baseline 改进、接口测试、并发测试和项目进度材料。
