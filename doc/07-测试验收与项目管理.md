# 07-测试验收与项目管理

## 1. 文档口径

本文档描述 Agent Censor 的目标测试验收和项目管理方案。当前内容为建议实现，不表示测试体系或项目管理流程已经完成。

## 2. 验收范围

验收应覆盖 `NOTE.md` 中的智能体、规则引擎、工作流、模型训练、多模态、RAG、API、前端、策略、高并发、自动化和项目管理要求。

## 3. 准确率评估

模型评估以准确率为核心指标，同时补充业务安全指标：

- 准确率：整体分类正确率。
- 召回率：高风险内容被识别的比例。
- 精确率：命中风险内容中真实违规的比例。
- 误杀率：正常内容被错误拦截或复核的比例。
- 漏放率：违规内容被错误放过的比例。
- baseline 改进：相对基线模型的提升，例如 95% + X。

高风险标签应优先关注召回率和漏放率，生态类风险应同时关注误杀率。

## 4. 测试类型

### 4.1 API 测试

- `POST /api/v1/moderation/tasks` 可提交单条审核任务。
- `POST /api/v1/moderation/batch` 可提交批量审核任务。
- `GET /api/v1/moderation/tasks/{task_id}` 可查询任务状态和结果。
- `POST /api/v1/policies/{policy_id}/rules` 可新增或更新规则。
- `GET /api/v1/labels` 可查询标签体系。

### 4.2 规则测试

- 关键词、正则、模型阈值、标签组合和 Graph RAG 节点条件均能命中。
- 策略版本切换后结果符合预期。
- 阈值调整后 `decision` 和 `suggested_action` 变化可解释。
- 禁用规则后不再影响审核结果。

### 4.3 模型测试

- 文本、图片、视频、音频、多模态模型分别评估。
- 模型路由能根据 `modality` 选择正确模型。
- 主模型不可用时能使用 `fallback_model`。
- 输出包含标签、分数和证据。

### 4.4 Graph RAG 测试

- 能检索政策、案例、规则和标签说明。
- 能返回命中节点、关系路径、相似度或置信度、证据摘要。
- 审核解释能引用 Graph RAG 证据。
- 不应生成无来源依据的政策解释。

### 4.5 树型智能体验收

- 能从 `config/settings.json` 的 `rules` 字段读取标签树。
- `rules.security` 能映射到 `SECURITY` 子树，`rules.ecosystem` 能映射到 `ECOSYSTEM` 子树。
- 叶子智能体命中时返回 `{ "label": "...", "reason": "..." }`，未命中时返回 `{ "label": "", "reason": "未检出问题" }`。
- 中间智能体返回 `{ "label": ["..."], "reason": "..." }`，并综合各个子智能体返回的 `reason`。
- 顶级智能体返回 `security_labels`、`ecosystem_labels` 和 `reason`。
- 父智能体不得修改、添加、删除收集到的子智能体标签，审计中应能复核该约束。

### 4.6 前端验收

- 审核结果常规情况下 3 秒内呈现。
- 命中规则高亮显示对应证据片段。
- 展示标签、风险等级、模型来源、规则来源、Graph RAG 解释。
- 支持人工复核一键通过、打回、改标和备注。
- 支持策略阈值调整结果预览。

### 4.7 并发测试

- 单条审核接口在并发请求下保持稳定。
- 批量审核接口能排队处理。
- 模型服务支持批处理或异步推理。
- 高并发下系统能限流、降级或返回明确状态。

## 5. 项目管理

项目建议按阶段推进：

| 阶段 | 目标产物 |
| --- | --- |
| P0 需求和设计 | 工程文档、标签体系、API 协议、架构图 |
| P1 最小可用审核链路 | 单条审核 API、基础规则、基础模型路由、审计记录 |
| P2 前端和规则配置 | 审核工作台、证据高亮、人工复核、策略预览 |
| P3 数据和 Graph RAG | 数据管理、RAG 知识库、图谱检索、解释增强 |
| P4 模型训练和评估 | baseline、专项模型、准确率评估、改进报告 |
| P5 部署和验收 | 部署脚本、监控告警、并发测试、验收报告 |

## 6. 进度管理

建议使用任务看板管理：

- 需求确认。
- 架构设计。
- API 开发。
- 规则引擎开发。
- 模型训练和部署。
- 前端工作台开发。
- 数据采集和清洗。
- Graph RAG 建设。
- 测试验收。
- 部署运维。

每个任务应记录负责人、状态、风险、依赖和验收标准。

## 7. 自动化

建议自动化内容：

- 自动运行 API 测试。
- 自动运行规则回归测试。
- 自动生成模型评估报告。
- 自动生成数据质量报告。
- 自动记录部署版本。
- 自动告警异常任务、模型超时、规则异常和数据采集失败。

## 8. 风险清单

| 风险 | 影响 | 缓解方案 |
| --- | --- | --- |
| 标签定义不清 | 模型和规则难以一致 | 在需求阶段固定标签编码和解释 |
| 数据质量不足 | 准确率和可解释性下降 | 建立来源、清洗、标注状态记录 |
| 前端只展示口号 | 演示效果弱 | 固定 3 秒展示、证据高亮、复核动作和策略预览 |
| Graph RAG 只有普通 RAG | 无法体现创新点 | 固定节点、关系路径和证据摘要输出 |
| 高并发下模型超时 | 用户体验下降 | 队列、批处理、缓存、降级 |
| 爬虫合规风险 | 交付和法律风险 | 仅采集授权或公开数据，不绕过访问控制 |
