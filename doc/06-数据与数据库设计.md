# 06-数据与数据库设计

## 1. 文档口径

本文档描述数据和数据库的目标设计。当前内容为建议实现，不表示数据集、数据库或爬虫系统已经完成。

## 2. 数据类型

目标系统需要管理以下数据：

- 审核样本数据：文本、图片、视频、音频和多模态样本。
- 爬虫数据：授权或公开来源采集的数据。
- 标注数据：人工标签、复核结果、争议标签。
- RAG 知识数据：政策条款、规则说明、典型案例、标签解释。
- embedding 数据：向量化后的政策、案例、规则和样本。
- 图谱数据：标签、政策、案例、规则、样本之间的节点和关系。
- 审计日志：请求、模型、规则、RAG 证据和处置结果。

## 3. 数据清晰度要求

每条样本记录至少应包含：

- `sample_id`：样本 ID。
- `source_type`：数据来源类型，例如公开数据、授权数据、人工构造、复核回流。
- `source_url` 或 `source_ref`：原始内容引用。
- `collected_at`：采集时间。
- `modality`：文本、图片、视频、音频或多模态。
- `raw_content_ref`：原始内容存储引用。
- `clean_status`：未清洗、已清洗、丢弃。
- `label_status`：未标注、已标注、复核中、争议。
- `labels`：标签列表。
- `usage_split`：训练、验证、测试、RAG、演示。
- `risk_note`：数据风险说明。

## 4. 合规爬虫数据

爬虫目标是补充公开或授权内容样本，不是规避平台访问控制。建议要求：

- 仅采集授权或公开数据。
- 尊重站点 robots、访问频率限制和服务条款。
- 使用限速、重试、断点续爬和失败告警。
- 记录来源、时间、采集脚本版本和清洗状态。
- 对重复内容、无效内容、低质量内容进行去重和过滤。
- 不设计验证码破解、登录绕过、Cookie 窃取、IP 池规避封禁等能力。

## 5. 数据集设计

数据集建议按任务和模态组织：

| 数据集 | 用途 |
| --- | --- |
| text_safety_dataset | 文本安全审核训练和评估 |
| image_safety_dataset | 图片色情、未成年等风险检测 |
| video_safety_dataset | 视频抽帧和时序风险检测 |
| audio_safety_dataset | 音频转写和语音风险检测 |
| multimodal_safety_dataset | 图文组合风险检测 |
| rag_policy_corpus | 政策、案例、规则和标签解释检索 |

每个数据集应明确训练、验证、测试划分，避免评估数据泄漏。

## 6. RAG 知识库

RAG 知识库应包含：

- 内容安全政策条款。
- 标签定义和适用范围。
- 典型案例。
- 规则说明。
- 业务策略说明。
- 模型误判和复核案例。

知识库文本应支持向量化，供语义检索使用。

## 7. Graph RAG 图谱设计

Graph RAG 需要图谱节点和关系。

建议节点类型：

- `Label`：标签。
- `Policy`：政策条款。
- `Case`：典型案例。
- `Rule`：规则。
- `Sample`：样本。
- `Business`：业务场景。

建议关系类型：

- `Label` -> `Policy`：标签依据哪些政策。
- `Policy` -> `Case`：政策对应哪些案例。
- `Case` -> `Rule`：案例沉淀为哪些规则。
- `Rule` -> `Label`：规则命中哪些标签。
- `Sample` -> `Label`：样本标注为哪些标签。
- `Business` -> `Policy`：业务使用哪些策略或政策。

检索结果必须支持命中节点、关系路径、相似度或置信度、证据摘要。

## 8. 数据库设计

建议数据库分层：

- 业务数据库：租户、业务、策略、规则、任务、结果、审计。
- embedding 数据库：政策、案例、规则、样本向量。
- 图谱数据库或关系表：节点和边。
- 对象存储：图片、视频、音频和原始样本。

核心表或集合建议：

- `tenants`：租户。
- `businesses`：业务线。
- `policies`：策略。
- `policy_rules`：规则；早期规则标签树入口来自 `config/settings.json` 的 `rules` 字段，后续可同步到数据库进行版本化管理。
- `moderation_tasks`：审核任务。
- `moderation_results`：审核结果。
- `audit_traces`：审计轨迹。
- `datasets`：数据集。
- `samples`：样本。
- `rag_documents`：RAG 文档。
- `graph_nodes`：图谱节点。
- `graph_edges`：图谱关系。
- `embeddings`：向量数据。

## 9. AuditTrace 设计

`AuditTrace` 应串联全链路：

- 请求快照。
- 规则标签树版本或配置来源，例如 `config/settings.json` 的 `rules` 字段。
- 树型智能体各级返回，包括叶子 `label/reason`、中间 `label[]/reason`、顶级 `security_labels/ecosystem_labels/reason`。
- 父智能体对标签不可修改、添加、删除的审计校验结果。
- 模型路由决策。
- 模型输出。
- 规则命中。
- Graph RAG 命中节点和路径。
- 最终结果。
- 人工复核动作。
- 时间、调用耗时、错误和降级信息。

审计数据用于项目验收、问题复盘、模型改进、规则优化和安全追责。
